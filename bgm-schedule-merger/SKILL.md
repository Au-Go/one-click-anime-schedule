# BGM 播出时间整合工具

从 bgm.wiki 获取动画的播出时间（日本时间）和播放平台信息，与已有的新番表文件整合，生成包含完整信息的汇总文件。

## 使用方式

```bash
# 基本用法
node .claude/skills/bgm-schedule-merger/merge.js --input "新番表路径" --output "输出文件路径"

# 指定季节（自动计算时间范围）
node .claude/skills/bgm-schedule-merger/merge.js --input "新番表路径" --season 春 --year 2026

# 指定时间范围
node .claude/skills/bgm-schedule-merger/merge.js --input "新番表路径" --from 2026-04-01 --to 2026-06-30
```

## 参数说明

| 参数 | 说明 | 必需 |
|------|------|------|
| `--input` | 已有新番表文件路径 | 是 |
| `--output` | 输出文件路径 | 否（默认: 输入文件名_bgm_info.txt） |
| `--season` | 季节（冬/春/夏/秋） | 二选一 |
| `--year` | 年份 | 否（默认: 2026） |
| `--from` | 开始日期（YYYY-MM-DD） | 二选一 |
| `--to` | 结束日期（YYYY-MM-DD） | 否（默认: from后3个月） |

## 季节定义

| 季节 | 时间范围 |
|------|----------|
| 冬 | 上一年12月 - 本年2月 |
| 春 | 3月 - 5月 |
| 夏 | 6月 - 8月 |
| 秋 | 9月 - 11月 |

## 输出格式

脚本会生成两个文件：

### 1. 已匹配文件（`*_bgm_info.txt`）
保留原有新番信息，在每条末尾紧跟添加播出信息，条目之间有空行分隔：

**输出示例：**
```
1. こんなこいるかな
   BGM ID: 635233
   类型: 泡面
   动画制作: 株式会社ライデンフィルム
   官方网站: https://konnako-irukana.com/
   放送日期: 2026年3月30日
   播出时间(JST): 07:45 (周一)
   播放平台: NHK-E

3. 春夏秋冬代行者 春の舞
   BGM ID: 551455
   中文名: 春夏秋冬代行者 春之舞
   类型: 小说改
   动画制作: WIT STUDIO
   官方网站: https://4seasons-anime.com/
   放送日期: 2026年3月28日
   播出时间(JST): 24:00 (周六)
   播放平台: TOKYO MX
```

### 2. 未匹配文件（`*_bgm_info_未匹配.txt`）
- 未在 bgm.wiki 找到播出信息的动画列表
- 已从主文件中移除，便于单独处理

## 匹配规则

### 优先级：bgmId 精确匹配

如果新番表中包含 `BGM ID` 字段，脚本会优先使用 bgmId 进行精确匹配：

```
1. Re:ゼロから始める異世界生活 4th season 喪失編
   BGM ID: 547888
   放送日期: 2026年4月8日
```

支持的格式：
- `BGM ID: 547888`
- `bgmId: 547888`
- `bgm.tv/subject/547888`
- `bangumi: 547888`

### 降级：标题模糊匹配

如果没有 bgmId 或匹配失败，会使用标题模糊匹配：

| 分数 | 匹配条件 | 说明 |
|------|----------|------|
| 100分 | 完全相等 | 标准化后完全一致 |
| 90分 | 日文标题包含 | 一方包含另一方 |
| 80分 | 中文标题包含 | 中文名匹配 |
| <70分 | 字符级匹配 | 计算字符重合比例 |

### 匹配率参考

| 匹配方式 | 匹配率 |
|----------|--------|
| bgmId 精确匹配 | ~83% |
| 标题模糊匹配 | ~77% |


## 工作原理

1. **解析输入文件**：读取已有的新番表，提取动画标题和 BGM ID
2. **获取 bgm.wiki 数据**：
   - 访问网站获取 API token
   - 按周分批获取播出表数据
   - 合并并去重
3. **去重逻辑**（选择播出时间）：
   - 按平台分组，每个平台选择最常见的播出时间（处理临时更改）
   - 选择当日最早播出的平台
4. **时间格式转换（30小时制）**：
   - bgm.wiki API 返回 UTC 时间（如 `T04:00:00Z`）
   - 加9小时转为 JST（如 `T13:00:00` → 13:00）
   - 深夜时段（JST 0:00~5:00）扩展为30小时制（25:00~30:00），星期回退一天
   - 示例：`T16:30:00Z` → JST 01:30 → 输出 `25:30 (前一天)`
5. **匹配数据**：
   - 优先使用 bgmId 进行精确匹配（匹配率 ~83%）
   - 降级使用标题模糊匹配（匹配率 ~77%）
6. **生成输出**：整合所有信息到一个文件

## 注意事项

- 数据来源：https://bgm.wiki/
- 播出时间显示为日本标准时间（JST，UTC+9），采用30小时制
  - bgm.wiki API 返回的 `eventAt` 为 ISO 8601 UTC 时间戳（如 `2026-04-06T04:00:00.000Z`）
  - 代码先将 UTC 转为 JST（+9小时），再将深夜时段（0:00~5:00）转换为30小时制（25:00~30:00）
  - 星期同步回退一天，与 bgm.wiki 页面显示一致（如 `25:30 (周五)` 表示周六凌晨1:30）
- 部分动画可能无法匹配（bgm.wiki 尚未收录或标题差异较大）
- 未找到信息的动画会标注"未找到"
